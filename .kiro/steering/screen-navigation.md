# screen-navigation.md — ビジ友（bijiyu）画面遷移

## 概要

ユーザー向け画面と管理者向け画面の遷移を定義する。ユーザー向けは認証系・受注者系・発注者系・共通系の4カテゴリ、管理者向けは管理者系の1カテゴリで構成。

---

## 1. 認証・ログイン前フロー

起点: AUTH-001（ログイン前TOP）

```
AUTH-001 ログイン前TOP
├→ AUTH-002 ログイン → CON-001 マイページ
│   └→ AUTH-003 パスワードリセット申請（※メール送信）
│       └→ AUTH-004 パスワード再設定 → 完了メッセージ表示 → AUTH-002 ログイン
├→ AUTH-005 新規会員登録メール認証（※認証メール送信）
│   └→ AUTH-006 新規会員登録情報入力
│       └→ AUTH-007 新規会員情報登録完了 → CON-001 マイページ（直接遷移）
├→ COM-007 よくある質問
├→ COM-008 お問い合わせ
├→ COM-009 利用規約
├→ COM-010 プライバシーポリシー
└→ COM-011 特定商取引法に基づく表示
```

### 備考
- ログイン成功後はCON-001（マイページ）へ遷移
- パスワードリセット申請で入力したメールアドレス宛にリセットURL送信
- **パスワード再設定完了後**: 完了メッセージ画面を表示し、ログイン画面（AUTH-002）へ遷移
- 新規会員登録はメール認証 → 情報入力 → 完了の3ステップ
- **新規登録完了後**: そのままマイページ（CON-001）に直接遷移（再ログイン不要）

## 2. 受注者系フロー（マイページ起点）

起点: CON-001（マイページ）

```
CON-001 マイページ
│
├→ CON-002 募集案件一覧
│   └→ CON-003 募集案件詳細
│       └→ CON-004 応募情報入力
│           └→ ポップアップ確認「この情報で応募して良いですか」
│               → OK → 完了ポップアップ表示（独立画面なし）
│
├→ CON-005 発注者一覧
│   └→ CON-006 発注者詳細
│
├→ CON-007 マイリスト
│   │  プルダウン切り替え:
│   │  - お気に入り案件一覧 → CON-003 募集案件詳細
│   │  - お気に入り発注者一覧 → CON-006 発注者詳細
│   │  （発注者のみ追加: 見込みユーザー一覧 → CLI-006 ユーザー詳細）
│
├→ CON-008 メッセージ/スカウト一覧（/messages）
│   │  1組織(or個人発注者)×1受注者＝常に1スレッド
│   │  法人プランは同一組織メンバー全員のスレッドを表示
│   │  受注者側: 相手が法人なら組織名、個人なら個人名を表示
│   └→ CON-009 メッセージ/スカウト詳細（/messages/{threadId}）
│       ├→ CON-010 メッセージ入力/送信（CON-009下部に内包）
│       └→ CON-004 応募情報入力（※スカウト受諾時: /jobs/{id}/apply?scout_message_id={id}）
│
├→ CON-011 応募履歴一覧
│   └→ CON-012 応募詳細
│       ├→ CON-013 作業報告・評価入力 →（送信後）CON-001 マイページ
│       └→ CON-003 募集案件詳細（「募集案件詳細」ボタン経由）
│
├→ CON-014 空き日程一覧
│   ├→ CON-015 空き日程更新
│   └→ CON-016 空き日程登録
│
├→ COM-003 本人確認・CCUS登録申請
│   ├→ COM-004 公的証明書・本人顔写真送付
│   └→ COM-005 CCUS技術者ID・本人確認番号入力
│
├→ COM-001 プロフィール詳細
│   ├→ COM-002 プロフィール編集
│   └→ COM-006 退会手続き
│
├→ CLI-026 有料プラン案内
│   ├→ CLI-027 決済画面（Stripe Checkout / 新規購入時のみ）
│   │   ├→ 成功（全プラン共通 / `individual` / `small` / `corporate` / `corporate_premium`）: CLI-021?setup=true（実 URL: `/mypage/client-profile/edit?setup=true`、発注者情報編集・初回設定モード）
│   │   │   ├─ 法人プラン: 社名必須 → 保存後 CON-001 へ
│   │   │   └─ 個人・小規模プラン: 任意、「スキップして後で設定する」ボタンを表示 → 押下で CON-001 へ（Webhook が client_profiles.display_name にデフォルト格納した姓名がそのまま表示名として使われる）
│   │   │   ※ billing 単独リリース時の暫定対応として法人プランのみ `/mypage/organization-setup` 経由、個人・小規模プランは `/mypage?checkout=success` に遷移。organization spec 実装時に両方削除され、全プランで CLI-021?setup=true に統合される
│   │   └→ キャンセル: CLI-026 に戻る
│   ├→ アップグレード: 確認ダイアログ → 即時反映（Stripe Checkout は経由しない）
│   │   └→ 全プランで CLI-021?setup=true へ自動遷移（暫定期間中は法人プランのみ /mypage/organization-setup 経由、個人・小規模プランはトースト表示のみ）
│   ├→ ダウングレード/解約: 前提条件チェック → 確認ダイアログ → 予約実行
│   │   └→ 予約キャンセル可能（請求期間終了日まで）
│   └→ Stripe Customer Portal（カード更新・請求履歴）
│
│   ※ プラン購入/アップグレード後の発注者情報設定（全プラン共通）:
│   CLI-021?setup=true に遷移 → 社名・氏名（client_profiles.display_name）入力 or スキップ（非法人のみ） → CON-001
│   CON-001 → CLI-020（発注者情報詳細）→ CLI-021（発注者情報編集）
│   法人プランのみ追加: CON-001 → CLI-022（担当者一覧）→ CLI-025（担当者新規作成）→ 招待メール送信
│                                      ↓（受信側）
│                                      メール内リンク → /auth/callback?type=invite → AUTH-008（/accept-invite/confirm）→ CON-001
│
├→ COM-007 よくある質問
├→ COM-008 お問い合わせ
├→ COM-009 利用規約
├→ COM-010 プライバシーポリシー
└→ COM-011 特定商取引法に基づく表示
```

### 備考
- メッセージ/スカウト一覧から「メッセージ一斉送信」「スカウト送信」への遷移は発注者専用機能のため、受注者にはボタンが表示されない
- 受注者はスカウトの受信・返信・受諾・拒否が可能。スカウト受諾時は CON-004（応募情報入力）に遷移する
- スカウト拒否後もメッセージの送受信は引き続き可能（ボタンのみ非表示）
- スカウトメッセージは通常メッセージと同じスレッド内にインライン表示される。受諾/拒否はメッセージ単位で管理
- 有料プラン案内から課金すると発注者機能が解放される
- **応募フロー**: 応募情報入力 → ポップアップ確認 → 完了ポップアップ（独立した確認/完了画面は存在しない）
- **応募詳細（CON-012）→ 募集案件詳細（CON-003）**: CON-012 に「募集案件詳細」カプセル型ボタンを設置し、対象案件の CON-003 に遷移可能とする
- **マイリスト**: プルダウンで切り替え。受注者は2項目（お気に入り案件/発注者）、発注者は3項目（+見込みユーザー）

## 3. 発注者系フロー（マイページ起点）

起点: CON-001（マイページ） ※ 発注者機能は課金後にマイページから追加メニューとしてアクセス可能

```
CON-001 マイページ（発注者メニュー追加）
│
│  ※ 発注者（client）は受注者系フロー（セクション2）の全画面にもアクセス可能。
│     CON-002 → CON-003 → CON-004 の遷移は有料の受注者と完全に同一。
│  ※ マイページの「仕事を探す」セクション（CON系画面への導線）は
│     発注者でもそのまま表示し、非表示にしてはならない。
│  ※ 「発注先を探す」セクションにCON系画面の導線を重複して含めないこと。
│  ※ 担当者（staff）はCON系画面の閲覧は可能（市場調査目的）。
│     ただし受注者アクション（応募・空き日程管理）は不可。
│     CON-003 の応募ボタンは非表示。CON-004/CON-011〜016 はMiddlewareでブロック。
│
├→ CLI-001 募集現場一覧
│   ├→ CLI-002 募集現場詳細
│   │   └→ CLI-003 募集現場編集 → 保存成功 → CLI-002（?manage=true）→「もどる」→ CLI-001
│   └→ CLI-004 募集現場新規登録 → 保存成功 → CLI-002（?manage=true）→「もどる」→ CLI-001
│
├→ CLI-005 ユーザー一覧（職人一覧）
│   └→ CLI-006 ユーザー詳細（職人詳細）
│       ├→ CLI-013 メッセージ詳細（/messages/new?to={userId} → /messages/{threadId} にリダイレクト）
│       │   ※ スレッドが存在しない場合は自動作成（相手の組織を検出して organization_id を設定）
│       ├→ CLI-015 スカウト送信（/messages/scout-send?userId={id}）
│       │   ※ 既存スレッドにスカウトメッセージを追加。なければ新規スレッド作成
│       └→ CLI-028 発注者評価
│
├→ CLI-007 応募一覧
│   └→ CLI-008 応募詳細
│       ├→ CON-003 募集案件詳細（「募集案件詳細」ボタン経由）
│       ├→ CLI-006 ユーザー詳細（「ユーザー詳細」ボタン経由）
│       └→ CLI-009 発注可否（「発注可否」ボタン経由、status='applied' のみ）
│           └→ ポップアップ「結果を送信しました」→ CLI-007 応募一覧へ遷移
│
├→ CLI-010 発注履歴一覧
│   └→ CLI-011 発注履歴詳細
│       └→ CLI-012 作業完了/失注報告・評価登録
│
├→ メッセージ/スカウト一覧（CON-008と共通、/messages）
│   │  発注者・担当者: 「一斉送信」ボタン表示
│   │  法人プラン: 組織メンバー全員のスレッドを表示
│   ├→ CON-009 メッセージ/スカウト詳細（/messages/{threadId}）
│   │   └→ CON-010 メッセージ入力/送信（CON-009下部に内包）
│   └→ CLI-014 メッセージ一斉送信（/messages/bulk-send、発注者専用）
│
├→ CLI-016 スカウトメッセージテンプレート一覧
│   ├→ CLI-017 スカウトメッセージテンプレート詳細
│   │   └→ CLI-018 スカウトメッセージテンプレート編集
│   └→ CLI-019 スカウトメッセージテンプレート新規作成
│
├→ CLI-020 発注者情報詳細
│   ├→ CLI-021 発注者情報編集
│   └→ CLI-022 担当者一覧
│       ├→ CLI-023 担当者詳細
│       │   └→ CLI-024 担当者編集
│       └→ CLI-025 担当者新規作成
│
└→ CLI-026 有料プラン案内
    └→ CLI-027 決済画面
```

### 備考
- 発注者メニューは課金後にマイページに追加される形（1アカウントで受注・発注の両方を利用可能）
- ユーザー詳細（職人詳細）からメッセージ・スカウト・評価の3つの導線がある。メッセージとスカウトは同じスレッドに入るが、入り口（UI）が異なる: 「メッセージを送る」= スレッド詳細に直行、「スカウトを送る」= 案件選択+テンプレート選択の専用フォームを経由
- 募集現場一覧から詳細→編集、または新規登録の2つのフロー。いずれも保存成功後は CLI-002（`?manage=true`）へ遷移し、CLI-002 の「もどる」ボタンで CLI-001 に戻る
- 担当者管理は発注者情報詳細の配下に位置
- **発注可否後のフロー**: ポップアップで「ユーザーへ結果を送信しました」を表示し、応募一覧（CLI-007）へ遷移。受注者へはメール通知 + 応募履歴のステータス更新
- **発注者マイページのメニュー構成ルール**: 発注者のマイページは「仕事を探す」セクション（受注者系フロー = CON系画面への導線）をそのまま維持し、その下に「発注先を探す」セクション（発注者専用フロー = CLI系画面への導線）が追加される形にすること。「仕事を探す」セクションを発注者で非表示にしたり、CON系の導線を「発注先を探す」に移動してはならない。これにより、発注者が CON-002（全案件の検索・応募画面）と CLI-001（自分の掲載案件の管理画面）を混同することを防ぐ
- **CON-003 と CLI-002 の区別**: CON-003（募集案件詳細 = 閲覧・応募用）と CLI-002（募集現場詳細 = 管理・編集用）は異なる画面だが、同じ案件のIDを参照する。URLパスが同一（`/jobs/[id]`）になる場合は、CLI-001 からのリンクに `?manage=true` クエリパラメータを付与し、CON-002 からのリンクにはパラメータなし（デフォルトで CON-003 を表示）とすることで区別する。発注者が CON-002 経由でアクセスした場合は常に CON-003（閲覧・応募画面）を表示すること

## 4. 共通系画面の遷移元

共通系画面は複数の画面から遷移可能。主な遷移元を整理する。

| 共通画面 | 主な遷移元 |

| COM-004 公的証明書・本人顔写真送付 | COM-003 本人確認・CCUS登録申請 |
| COM-005 CCUS技術者ID・本人確認番号入力 | COM-003 本人確認・CCUS登録申請 |
| COM-006 退会手続き | COM-001 プロフィール詳細 |
| COM-007 よくある質問 | AUTH-001 ログイン前TOP / CON-001 マイページ |
| COM-008 お問い合わせ | AUTH-001 ログイン前TOP / CON-001 マイページ |
| COM-009 利用規約 | AUTH-001 ログイン前TOP / CON-001 マイページ |
| COM-010 プライバシーポリシー | AUTH-001 ログイン前TOP / CON-001 マイページ |
| COM-011 特定商取引法に基づく表示 | AUTH-001 ログイン前TOP / CON-001 マイページ |

## 5. 管理者系フロー

起点: ADM-001（管理者ログイン）→ ADM-002（管理者トップページ）

```
ADM-001 管理者ログイン
└→ ADM-002 管理者トップページ
    │
    ├→ ADM-003 発注者アカウント一覧
    │   ├→ ADM-004 発注者アカウント詳細
    │   │   └→ ADM-005 発注者アカウント編集
    │   └→ ADM-006 発注者 管理責任者 新規作成
    │       └→ ADM-007 発注者 管理責任者 新規作成確認
    │
    ├→ ADM-008 ユーザーアカウント一覧
    │   └→ ADM-009 ユーザーアカウント詳細
    │       └→ ADM-010 ユーザー動画投稿（TikTok動画の代理投稿、動画掲載オプション加入者のみ）
    │
    ├→ ADM-011 本人確認承認申請一覧
    │   └→ ADM-012 本人確認承認可否
    │       → 承認/否認後: ユーザーへメール通知 + マイページのステータスバッジ更新
    │
    ├→ ADM-013 応募履歴一覧
    │   └→ ADM-014 応募履歴詳細
    │       ├→ CLI-002 募集現場詳細（案件情報クリック）
    │       └→ ADM-009 ユーザーアカウント詳細（ユーザー情報クリック）
    │
    └→ ADM-015 管理者パスワード変更
```

### 備考
- 管理者ログインは一般ユーザーとは完全に別の画面
- 応募履歴詳細から募集現場詳細・ユーザーアカウント詳細へのクロスリンクあり
- 発注者アカウント詳細で「アカウントを削除する」→ 確認ポップアップ（画面遷移なし）
- ユーザーアカウント詳細で「アカウントを削除する」→ 確認ポップアップ（画面遷移なし）
- **ユーザー動画投稿**: 管理者がTikTok動画を代理投稿する機能。動画掲載オプション加入者のみ表示
- **本人確認承認/否認**: メールで通知 + マイページの本人確認・CCUS登録エリアにステータスバッジを表示（次回アクセス時に反映）

## ポップアップ・モーダル一覧

画面遷移を伴わないポップアップ・モーダルの一覧。

| 発生元画面 | ポップアップ内容 | OK後の遷移先 |
|-----------|---------------|-------------|
| CON-004 応募情報入力 | 「この情報で応募して良いですか」確認 → OK → 「応募が完了しました」完了表示 | CON-011 応募履歴一覧 |
| CLI-009 発注可否 | 「ユーザーへ結果を送信しました」完了表示 | CLI-007 応募一覧 |
| COM-006 退会手続き | 退会確認ポップアップ → 退会完了メール送信 | AUTH-001 ログイン前TOP（セッション無効化後リダイレクト） |
| ADM-004 発注者アカウント詳細 | アカウント削除確認ポップアップ（管理責任者削除で配下担当者も削除） | ADM-003 発注者アカウント一覧 |
| ADM-009 ユーザーアカウント詳細 | アカウント削除確認ポップアップ | ADM-008 ユーザーアカウント一覧 |
| CLI-023 担当者詳細 | 担当者削除確認ポップアップ | CLI-022 担当者一覧 |
| CON-009 メッセージ/スカウト詳細 | 「スカウトを断りますか？」確認ダイアログ → 断る → ボタン非表示（画面遷移なし） | 同一画面（ボタンのみ非表示、メッセージ送受信は継続可能） |
