# 案件掲載機能（job-posting）— 要件定義

## 概要

発注者が募集現場（案件）を作成・管理する機能。案件の新規登録・編集・一覧表示・詳細表示を提供する。

## 対象画面

| 画面ID | 画面名 | 概要 |
|--------|--------|------|
| CLI-001 | 募集現場一覧 | 自社の掲載案件リスト |
| CLI-002 | 募集現場詳細 | 案件の詳細情報・応募者管理 |
| CLI-003 | 募集現場編集 | 既存案件の編集フォーム |
| CLI-004 | 募集現場新規登録 | 新規案件の作成フォーム |

## 対象ロール

- 発注者（Client）: 個人プラン — 1件/月、その他有料プラン — 無制限
- 担当者（Staff）: 所属組織の案件を操作可能
- 管理責任者/管理者: 組織内の全案件を操作可能

※ 受注者（無料）はアクセス不可

## 機能要件

### REQ-JP-001: 募集現場一覧（CLI-001）

- 自分（または所属組織）が作成した案件の一覧を表示する
- 表示項目: タイトル、職種、エリア、報酬、ステータス、募集期間
- ステータス: 下書き（draft）/ 募集中（open）/ 締切済（closed）
- ソート: 作成日時の降順（デフォルト）
- ページネーション: 20件ずつ
- 「新規登録」ボタン → CLI-004 へ遷移
- 各案件クリック → CLI-002 へ遷移
- 法人プランの場合: 組織内の全案件を表示（作成者名を付記）

### REQ-JP-002: 募集現場詳細（CLI-002）

- 案件の全情報を表示する
- 表示スタイルは CON-003（募集案件詳細）と統一する（DetailRow 形式 + セクション分割）
- レイアウト構成（上から順に）:
  1. ヘッダー（「募集現場詳細」+ 掲載中の場合「掲載を終了する」ボタン。押下時に確認ダイアログを表示し、ステータスを closed に更新する）
  2. ステータスバッジ + 急募バッジ（オプション加入時のみ）
  3. 添付画像（job_images）。画像がない場合はプレースホルダーを表示
  4. タイトル + 会社名 + 案件詳細（description）
  5. アクションボタン（「応募者をみる」「編集する」— 同幅・中央配置）
  6. 条件セクション — DetailRow 形式（alwaysShow）で以下を表示:
     - 報酬、エリア、住所、募集職種、募集人数、現場工期、募集期間、稼働時間、締め切り、経験年数、必須スキル、国籍・言語、持ち物
  7. 業務内容セクション — スケジュール詳細、請負案件詳細
  8. 発注者からのメッセージセクション — 枠付きカード表示
  9. アクションボタン（下部にも「応募者をみる」「編集する」を配置 — 同幅・中央配置）
  10. 「コピーして新規作成する」ボタン（primary カプセル型）と「もどる」ボタン（outline カプセル型）— 同幅・縦並び中央配置
- 値が空（null/未入力）の項目も項目名と「—」を表示する（非表示にしない）
- 「編集する」ボタン → CLI-003 へ遷移
- 「応募者をみる」ボタン → CLI-007（応募一覧）への導線

### REQ-JP-003: 募集現場編集（CLI-003）

- 既存案件の内容を編集する
- 編集可能フィールド: REQ-JP-004 の入力項目と同一
- 現在の登録内容をフォームにプリフィルする
- ステータス変更: draft → open → closed の遷移が可能
- 画像の追加・削除・並び替えが可能
- バリデーション: 公開時は jobSchema（全必須項目チェック）、下書き保存時は jobDraftSchema（タイトルのみ必須）
- ボタン構成: draft 状態 → 「公開する」+「下書き保存」、非draft 状態 → 「更新する」
- 保存成功時: CLI-002（詳細）へ遷移

### REQ-JP-004: 募集現場新規登録（CLI-004）

- 新規案件を作成する
- コピー機能: CLI-002 の「コピーして新規作成する」ボタンから遷移した場合（`?copyFrom={jobId}`）、コピー元の案件データをフォームにプリフィルする
  - コピーされるフィールド: タイトル、案件詳細、職種、報酬、勤務地、募集人数、稼働時間、経験年数、スキル、国籍・言語、持ち物、スケジュール詳細、請負案件詳細、発注者メッセージ
  - コピーされないフィールド: 工期（開始日・終了日）、募集期間（開始日・終了日）、画像・書類
  - ステータスは常に 'draft' で初期化する
- 入力フィールド:
  - タイトル（必須）
  - 案件詳細（必須）
  - 職種（必須、OptionSets から選択）
  - 報酬下限（必須、数値）
  - 報酬上限（必須、数値、下限以上）
  - 勤務地 — 都道府県（必須、OptionSets から選択）
  - 勤務地 — 詳細住所（任意）
  - 現場工期 — 開始日・終了日（必須）
  - 募集期間 — 開始日・終了日（必須）
  - 募集人数（必須、数値）
  - 稼働時間（任意）
  - 必要経験年数（任意）
  - 必須スキル（任意）
  - 国籍・言語（任意）
  - 持ち物（任意）
  - スケジュール詳細（任意）
  - 請負案件詳細（任意）
  - 発注者からのメッセージ（任意）
  - 画像・書類アップロード（任意、複数可、photo / document タイプ）
- 初期ステータス: 'draft'（下書き）
- 個人プラン（¥3,800）の場合: 当月の掲載件数が1件以上なら作成不可（エラーメッセージを表示）
- 「当月」の定義: **カレンダー月（JST基準）**（1月1日〜1月31日、2月1日〜2月28日…）。毎月1日の JST 0:00 にカウントがリセットされる。Supabase の timestamptz は UTC 保存のため、カウント時に JST に変換する。参考SQL: `WHERE created_at >= date_trunc('month', NOW() AT TIME ZONE 'Asia/Tokyo') AT TIME ZONE 'Asia/Tokyo'`
- jobs テーブルおよび job_images テーブルにデータを保存
- 法人プランの場合: organization_id を自動設定
- 保存成功時: CLI-002（詳細）へ遷移

## 非機能要件

### セキュリティ

- Middleware: 発注者（課金済み）のみアクセス可能
- RLS: 自分（または所属組織）の案件のみ読み書き可能
- 画像アップロード: JPEG, PNG のみ、最大10MB/枚
- 画像はファイル名をUUIDに変換して Supabase Storage（job-attachments バケット）に保存
- 全フォーム入力に Zod バリデーション（クライアント + サーバー）

### プラン制限の検証

- 掲載件数制限は Server Action で検証する（フロントエンドのみでの制御は不可）
- 個人プラン: 当月（カレンダー月）のステータスが open かつ deleted_at IS NULL の案件数をカウント。下書き（draft）や削除済みの案件はカウントに含めない

## 画面遷移

```
CON-001（マイページ）→ CLI-001（一覧）→ CLI-002（詳細）→ CLI-003（編集）
                                      → CLI-004（新規登録）→ CLI-002（詳細）
```
